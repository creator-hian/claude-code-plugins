# Examples Directory

Detailed examples for each best practice principle.

## Core Principles (1-12)

1. [Keep It Simple](01-keep-it-simple.md) - Concise summaries for straightforward elements
2. [Add Context Where Needed](02-add-context.md) - Detailed explanations for complex concepts
3. [Think About Your Audience](03-language-choice.md) - Choose language that serves your team
4. [Be Consistent](04-consistency.md) - Follow established patterns
5. [Avoid Redundancy](05-avoid-redundancy.md) - Don't repeat obvious information
6. [Use Structured XML](06-structured-xml.md) - Leverage XML tags for clarity
7. [Document Special Cases](07-special-cases.md) - Explain magic numbers and unusual behaviors
8. [Document Exceptions](08-exceptions.md) - Use `<exception>` for contract exceptions
9. [Document Enums Completely](09-enums.md) - Clear description for each enum value
10. [Extension Methods](10-extension-methods.md) - Document the extended type
11. [Property Side Effects](11-property-side-effects.md) - Use `<value>` for non-obvious behavior
12. [Interface vs Implementation](12-interface-implementation.md) - Full docs in interface, `<inheritdoc/>` in implementation
